# Melhorias recomendadas

## Prioridade alta

- Vincular os canais comerciais oficiais da Nexus7 no `README.md`.
- Publicar mais dois ou tres projetos reais com resumo executivo e estudo de caso.
- Padronizar capturas de tela, mockups e evidencias visuais em `assets/`.
- Corrigir o nome publico do repositorio para remover o erro de grafia `portifolio`, que passa amadorismo.
- Fixar no GitHub os repositorios mais fortes para alinhar a vitrine da organizacao com o posicionamento do estudio.

## Prioridade media

- Adicionar versao resumida em ingles do `README.md` para uso em Fiverr.
- Criar modelos de proposta tecnica e checklist de descoberta comercial.
- Definir um padrao de "antes e depois" para projetos de reestruturacao.
- Adicionar mais links entre projeto, estudo de caso e codigo publico.

## Prioridade baixa

- Criar uma pagina publica futura com base no mesmo posicionamento deste repositorio.
- Automatizar verificacao de links do README e da documentacao.
- Evoluir o banner e a identidade visual com um kit de marca completo quando houver maturidade suficiente.

## Melhorias de codigo e organizacao

- Evitar arquivos grandes de midia na vitrine publica quando nao agregarem valor direto.
- Priorizar repositorios com README forte, setup claro e objetivo de negocio visivel.
- Manter a mesma estrutura para todos os projetos publicados.
- Remover linguagem vaga e claims nao verificaveis de qualquer material publico.
